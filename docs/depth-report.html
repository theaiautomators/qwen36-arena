How this was measured (and what's honest)

  • Same GPU, same prompts, one client-side stopwatch for both engines · greedy · thinking off · 256 tokens · warmup discarded · n=5 median · every output length-checked for coherence (zero garbage runs).
  • The ranking is triple-confirmed — the depth sweep, a back-to-back re-measure, and a clock-locked run all put NVFP4-tuned ahead by ~1.2–1.3× (on both the code and math tasks). llama.cpp's own server timer agrees with our stopwatch to within 0.4%.
  • Exact peak wobbles ±~5–8% (NVFP4 code ~130–140) — it's vLLM engine warm-up, not GPU boost (locking the clocks didn't remove it; more warm-up requests do). The math number and the ranking are rock-solid.
  • The "2.5× faster" headline is a different axis — that's batched throughput on a data-centre B200 (128 users at once). Solo, on one card, it's a wash until you turn MTP depth up — which is the whole point of this chart.
